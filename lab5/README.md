# Лабораторная работа №5

## Тема

Разработка параллельных программ с использованием OpenMP.
Индивидуальный вариант №1: **произведение матриц A и B**.

## Постановка задачи

Даны произвольные квадратные матрицы `A` и `B` одинаковой размерности.
Реализовать вычисление произведения двумя способами:

1. **Способ 1.** Количество потоков задаётся как входной параметр; размерность
   матриц может быть не кратна количеству потоков.
2. **Способ 2.** Количество потоков заранее неизвестно и не является
   параметром задачи (используется значение OpenMP по умолчанию).

## Структура

- `Matrix` — простой класс матрицы с плотным хранением `std::vector<double>`.
- `MultiplySerial` — последовательное произведение, эталон для проверки.
- `MultiplyParallelFixedThreads(a, b, threads)` — способ 1, явный
   `num_threads(threads)` в директиве `#pragma omp parallel for`.
- `MultiplyParallelAuto(a, b)` — способ 2, без указания числа потоков.

В обоих параллельных вариантах используется `collapse(2)` (только в первом
варианте — внешний цикл и так выровнен по строкам; во втором варианте оставлен
один уровень распараллеливания, чтобы продемонстрировать различие подходов).

## Запуск

```bash
cmake -S . -B build
cmake --build build

./build/matmul_app serial   512
./build/matmul_app parallel 512 4    # способ 1
./build/matmul_app auto     512      # способ 2
./build/matmul_app bench    1024     # таблица замеров
./build/matmul_tests
```

В Docker:

```bash
docker build -t lab5 .
docker run --rm lab5
```

## Анализ

- **Сложность последовательного алгоритма**: `O(n^3)`.
- **Параллельная сложность на p потоках**: `O(n^3 / p)` при идеальном балансе.
- **Ускорение** `S(p) = T(1) / T(p)`.
- **Эффективность** `E(p) = S(p) / p`.
- **Стоимость** `C(p) = p * T(p)`.

В режиме `bench` программа печатает таблицу со временем последовательной
и параллельной (1, 2, 4 потока) реализаций и наблюдаемым ускорением.

## Тесты

`matmul_tests` проверяет:

- умножение на единичную матрицу даёт исходную матрицу;
- известный пример `2x3 * 3x2 = 2x2` для всех трёх реализаций;
- параллельные реализации совпадают с последовательной при размерах,
  не кратных количеству потоков (7×7 на 3, 4 и 8 потоках);
- несовпадение форм бросает `std::invalid_argument`.
