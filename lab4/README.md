# Лабораторная работа №4

## Тема

Применение OpenMP к алгоритму релаксации Якоби в трёхмерной сетке.
Сравнение последовательной и параллельной версий по времени и ускорению.

## Алгоритм

В кубе `N x N x N` точки на границе зануляются, во внутренних узлах задаётся
начальное распределение. На каждой итерации:

1. **relax** — вычисление обновлённых значений `B[i][j][k]` как среднего шести
   соседей по `A`.
2. **resid** — копирование `B` в `A` и вычисление максимальной невязки `eps`.

Цикл идёт до достижения `itmax` или пока `eps < maxeps`.

## Запуск

```bash
cmake -S . -B build
cmake --build build

# Последовательная версия
./build/jacobi_app serial 128 12

# Параллельная версия (2 потока)
./build/jacobi_app parallel 128 12 2

# Полная таблица для k = номер по списку
./build/jacobi_app table 10

# Тесты
./build/jacobi_tests
```

В Docker:

```bash
docker build -t lab4 .
docker run --rm lab4
```

## Закон Амдала

Если доля последовательного кода `s`, а параллельного `1 - s`, то теоретическое
ускорение на `p` ядрах ограничено `1 / (s + (1 - s) / p)`. На практике релакс и
резидум хорошо распараллеливаются по `i, j`, и наблюдаемое ускорение на 2 ядрах
близко к 2, но не превышает теоретического предела из-за накладных расходов на
создание потоков и синхронизацию reduction.

## Тесты

`jacobi_tests` проверяет, что:

- инициализация выдаёт ненулевую контрольную сумму;
- одна итерация даёт положительный `eps`;
- параллельная версия даёт ту же контрольную сумму, что и последовательная.
