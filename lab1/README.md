# Лабораторная работа №1

## Тема

Сравнение алгоритмов сортировки на одинаковых входных данных.

## Реализованные алгоритмы

| Алгоритм        | Балл | Сложность (среднее) | Сложность (худший) |
|-----------------|------|---------------------|--------------------|
| Bubble Sort     | 3    | O(n²)               | O(n²)              |
| Insertion Sort  | 5    | O(n²)               | O(n²)              |
| Quick Sort      | 10   | O(n log n)          | O(n²)              |

**Сумма баллов: 18.**

Все три алгоритма реализованы как наследники общего интерфейса
`ISorter<T>` — это позволяет единообразно замерять время и подменять
реализации в `CompareSorters`.

## Структура

```
lab1/
├── include/
│   ├── ArraySequence.h     — массив с операциями Get/Set/Append/Print

│   ├── InsertionSorter.h   — сортировка вставками

├── src/main.cpp            — интерактивное меню
├── tests/test_sort.cpp     — юнит-тесты для всех сортировщиков
├── CMakeLists.txt
└── Dockerfile
```

## Возможности приложения

- ручной ввод последовательности;
- генерация случайной последовательности (`std::mt19937`,
  равномерное распределение в `[0; 100000]`);
- запуск каждой из трёх сортировок по отдельности;
- режим сравнения, печатающий время выполнения всех трёх алгоритмов
  на одних и тех же данных;
- печать текущей последовательности;
- автоматические тесты для проверки корректности.

## Особенности и нюансы

- **Полиморфизм через `ISorter<T>`.** Функция `MeasureSortTime` принимает
  `ISorter<T>&`, поэтому при добавлении новой сортировки достаточно
  реализовать класс-наследник; код измерения и сравнения не меняется.
- **Bubble Sort без оптимизации.** Не используется ранний выход при
  отсутствии обменов — это сделано намеренно, чтобы получить честный
  верхний край асимптотики и заметную разницу с Quick Sort на больших
  размерах.
- **Quick Sort с pivot по центру.** `pivot = sequence[(left + right) / 2]`
  — это разумный компромисс: даёт `O(n log n)` на уже отсортированных
  и обратно отсортированных данных (тогда как классический выбор
  pivot = `arr[left]` на них деградирует до `O(n²)`).
- **Тесты покрывают граничные случаи.** Пустая последовательность,
  один элемент, уже отсортированная, обратно отсортированная,
  с дубликатами — всё это в `TestSorter`.
- **Bench-сравнение через `std::chrono`.** Точность измерений — миллисекунды;
  для малых размеров (`n < 1000`) Bubble и Insertion могут показать `0 ms` —
  это норма для такого таймера.

## Сборка и запуск

```bash
cmake -S . -B build
cmake --build build

./build/sort_app          # интерактивное меню
./build/sort_tests        # запуск автотестов
```

В Docker:

```bash
docker build -t lab1 .
docker run --rm lab1
```
